## 一、项目概述

### 1.1 项目背景

山东出版供应链管理公司业务平台，是面向文旅行业的 **企业级B2B业务管理系统**，集供应链管理、合同审批、财务管控、票务代理销售于一体。

**核心业务范围：**
1. **供应链管理** — 景区/文旅甲方档案管理、供应商资质管控
2. **合同审批** — 7级审批流，支持付款审批单/业务审批单
3. **财务管控** — 进账出账、结算付款、发票管理、银行对账
4. **票务代理** — 对接美团、抖音等OTA平台，实现多渠道票务销售与统一管理

**核心业务场景：**

```
┌─────────────────────────────────────────────────────────────────────┐
│                         B2B票务业务流程                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   甲方（景区/文旅） ──授权票种+密钥──► 出版社 ──绑定票种+密钥──► 美团 │
│        │                                    │                       │
│        │ 结算、发票                         │ 销售、结算             │
│        ◄────────────────────────────────────                        │
│                                                                     │
│   甲方（景区/文旅） ──授权票种+密钥──► 出版社 ──绑定票种+密钥──► 抖音 │
│        │                                    │                       │
│        │ 结算、发票                         │ 销售、结算             │
│        ◄────────────────────────────────────                        │
│                                                                     │
│   【核心模块】                                                       │
│   1. 合同管理：7级审批流，合规管控                                  │
│   2. 票务对接：美团/抖音OTA平台实时同步                            │
│   3. 财务管理：进账出账、结算付款、发票管理                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 交付范围

本项目交付包含两个终端：

| 终端 | 技术栈 | 说明 |
|------|--------|------|
| **Web管理端** | Vue3 + Element Plus | 面向内部员工的B2B管理系统，包含所有业务功能 |
| **小程序端** | 微信小程序 | 面向审批人员的移动审批助手，支持审批、查看等功能 |

> 说明：Web端与小程序端共用同一套后端API，数据实时同步。

| 序号 | 角色 | 职责 |
|------|------|------|
| 1 | 业务经办 | 创建合同、提交审批 |
| 2 | 业务复核 | 业务层面复核 |
| 3 | 风控审核 | 风险控制审核 |

| 7 | 投资公司负责人 | 最终审批 |

## 四、技术标准

### 4.1 工业级标准

本系统按照企业级B2B系统的工业标准进行开发，符合以下规范：

| 标准类别 | 具体要求 | 实现方式 |
|----------|----------|----------|
| **代码规范** | PEP8（Python）、ESLint+Prettier（前端） | 自动化检查、CI/CD |
| **API设计** | RESTful规范、OpenAPI 3.0文档 | 自动生成接口文档 |
| **安全认证** | JWT Token、基于角色的访问控制 | 细粒度权限控制 |
| **数据安全** | 敏感数据加密存储、SQL注入防护 | bcrypt加密、参数化查询 |
| **传输加密** | 前后端数据传输加密 | RSA+AES混合加密 |
| **XSS防护** | 输入过滤、输出转义 | DOMPurify + 后端过滤 |
| **CSRF防护** | Token验证 | 防CSRF攻击 |
| **日志审计** | 操作日志、登录日志 | 完整审计追踪 |
| **接口限流** | 防暴力破解、防DDoS | Redis限流、IP封禁 |
| **异常处理** | 全局异常捕获、统一响应格式 | 中间件统一处理 |

### 4.2 业务合规

| 合规要求 | 说明 |
|----------|------|
| **数据一致性** | 多平台数据统一口径，避免数据冲突 |
| **对账机制** | 支持平台账单与系统记录自动核对 |
| **审批留痕** | 关键操作（结算、付款）需审批通过 |
| **财务合规** | 进账出账有据可查，支持财务审计 |
| **密钥安全** | 平台API密钥加密存储，不明文展示 |
| **敏感数据保护** | 手机号、身份证等敏感信息脱敏展示 |

## C.3 「已接 DeepSeek 但回答质量不高」——上下文缺失根因分析

**结论先行**：模型本身没问题，问题在**喂给它的 Context 太贫瘠**。当前 `ai_agent._build_user_prompt()` 只塞了 **8 个高度汇总的标量**（营收、成本、利润、利润率、订单数、估算闲置资金、待开票额、审批中合同数），且其中数项是**估算或空值**。信息量不足 → 模型只能产出通用、模板化、放之四海皆准的建议，无法给出「针对本公司、可落地」的洞察。

### 具体缺了哪些上下文（按影响排序）
1. **时间趋势缺失**：只有本年**总量**，没有逐月序列、环比/同比。模型无法判断「在恶化还是好转」，给不出趋势性预警。（数据其实已有——`dashboard` 的 `trend` 逐月数据没喂给 AI。）
2. **业务结构缺失**：没有分业务线(`line_share`)的营收/利润构成，模型说不出「哪条线在亏、哪条线该加投」。（数据已有，未注入。）
3. **项目级明细缺失**：`biz_project_metrics` 里每个项目的投入/回款/收益率/资金占用/城市**完全没进 Context**。所以 AI 说不出「泉州项目回款慢、资金占用高」这类具体结论——而这恰是公司最关心的。
4. **平台维度缺失**：抖音/美团/携程对账单的分平台毛收入、订单、间夜未注入。
5. **真实资金头寸失真**：`idle_funds` 是 `利润×0.6` 拍脑袋值，`available_funds` 手工且常为 0。**投资建议的金额基于假数**，专业性和可信度大打折扣。
6. **应收账龄缺失**：`pending_invoice` 只有一个总额，没有账龄分布(30/60/90 天)，无法做回款风险分级。
7. **口径未对齐 / 可能喂空数据**：喂给 AI 的 `revenue/margin` 来自 `OperationData`(渠道回传口径)，而项目 `profit_rate` 来自 Sheet2(加权口径)，两套口径并存且未说明；若渠道未回传则 `revenue=0 → margin=0`，模型基于空数据臆测。
8. **无基准/无记忆/无检索**：单轮 prompt，无行业基准、无历史对比、无多轮追问、无工具调用(RAG)，`temperature=0.4` + 强制 JSON 使输出偏保守模板化；风险「利润率<30%」判断实质仍是规则式阈值，未发挥模型推理。

### 改进方向（不改模型，只补 Context 与工程）
- **把已有但没喂的数据注入 prompt**：逐月 `trend`、分线 `line_share`、分平台 `platforms`、Top-N 项目明细(投入/回款/收益率/资金占用/城市) —— 这几项**零新增数据成本**，是性价比最高的一步。
- **补真实资金口径**：用真实可用资金替换 `利润×0.6`；说明各指标口径与统计范围。
- **结构化 + 分段**：把「经营诊断」与「资金投资建议」拆成两次调用或分段 prompt，各自给足对应明细。
- **引入历史与基准**：存历史快照做同比/环比；给出行业毛利率基准区间供模型对照。
- **进阶(可选)**：RAG 检索合同/对账明细、多轮追问、Function-calling 让模型主动查数；流式输出改善大屏打字机体验。
